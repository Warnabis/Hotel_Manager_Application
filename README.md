# Hotel_Manager_Application

**Hotel Manager Application** — это приложение для управления гостиницей, предоставляющее интерфейс для работы с гостями, номерами, бронированиями, сотрудниками и их должностями, услугами и платежами.

## Этапы выполнения

#### 1.2. Расчет оценки времени

**Формула:**  
`E = (P + O + 4 × BG) / 6`

где:
- **P** — пессимистичная оценка
- **O** — оптимистичная оценка
- **BG** — наиболее вероятная оценка

**Исходные данные:**

| Параметр | Значение (часы) |
|----------|-----------------|
| Оптимистичная оценка (O) | 2 |
| Наиболее вероятная (BG) | 3 |
| Пессимистичная оценка (P) | 5 |

**Расчет:**
`E = (5 + 2 + 4 × 3) / 6 = 3.17 (≈ 3.2 часа)`

**Результат:** ожидаемое время выполнения этапа — **3.2 часа**.
**Фактически затраченно время:** — **2.5 часа**.

### 3. Разработка Backend на Spring Boot + Hibernate

#### 3.1. Постановка задачи

| Задача | Описание |
|--------|----------|
| Инициализация проекта | Создание Spring Boot проекта с необходимыми зависимостями (Spring Web, Spring Data JPA, PostgreSQL Driver, Lombok, Map, Validation) |
| Настройка подключения к БД | Конфигурация `application.properties` для подключения к PostgreSQL, настройка Hibernate (ddl-auto, диалект, логирование SQL) |
| Создание JPA-сущностей | Описание всех 7 сущностей с использованием аннотаций `@Entity`, `@Table`, `@Id`, `@GeneratedValue`, `@Column`. Настройка связей (`@OneToMany`, `@ManyToOne`, `@ManyToMany`, `@JoinTable`, `@JoinColumn`) |
| Создание репозиториев | Создание интерфейсов, расширяющих `JpaRepository` с использованием Spring Data JPA |
| Реализация бизнес-логики | Создание `@Service` классов с методами для CRUD-операций |
| Создание DTO и мапперов | Использование Map для преобразования Entity ↔ DTO |
| Создание REST-контроллеров | Реализация `@RestController` с эндпоинтами для всех операций. Поддержка пагинации и фильтрации с помощью `Pageable` и `@RequestParam` |
| Глобальная обработка ошибок | Создание `@RestControllerAdvice` для централизованной обработки исключений (ResourceNotFoundException, BadRequestException, ConflictException) |
| Валидация входных данных | Использование `@Valid` и Jakarta Validation для проверки запросов |
| Исправление замечаний SonarCloud | Анализ кода и устранение выявленных проблем |
